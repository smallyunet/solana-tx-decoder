# Community Guidelines

We aim to build a welcoming, thoughtful, and professional community around Solana transaction decoding.
These guidelines describe how we collaborate, review changes, and communicate.

## Be Respectful and Constructive

- Use clear, professional language and assume good intent.
- Keep discussions focused on the problem and the proposed solution.
- Provide feedback that is specific and actionable.

## Keep Work Transparent

- Share context when proposing changes or reporting issues.
- Link to relevant transactions, logs, or reproduction steps.
- Document decisions that affect public APIs or behavior.

## Contribute With Care

- Small, well-scoped pull requests are easier to review.
- Favor readability and explicitness over cleverness.
- Follow the established style in the codebase.

## Follow Project Policies

- For contribution details, see CONTRIBUTING.md.
- For expected behavior and enforcement, see CODE_OF_CONDUCT.md.

Thank you for helping make this project better.
